# Team Project Submission Guide

Use this checklist before submitting your team project.

## 1. GitHub repository

- [ ] Create a GitHub repository for the team project.
- [ ] Add all team members as collaborators.
- [ ] Add Stelios as a collaborator using this email address: `steliosot@msn.com`.
- [ ] Push the final code, data files, reports, and README to GitHub.
- [ ] Do not push your `.venv` folder to GitHub.
- [ ] Do not include API keys, secrets, or unrelated temporary files.

## 2. Team video

- [ ] Record the team presentation video.
- [ ] One student from the team uploads the video to OneDrive.
- [ ] Share the OneDrive video with `s.sotiriadis@bbk.ac.uk`.
- [ ] Include the required video link in the peer evaluation form.

## 3. Peer evaluation form

Each team member must complete and submit the peer evaluation form individually.

- [ ] Every team member completes the peer evaluation form.
- [ ] Every team member submits the peer evaluation form in Moodle.
- [ ] Use the provided [Peer_Evaluation_Form.docx](./Peer_Evaluation_Form.docx).

> [!IMPORTANT]
>
> **If a student does not submit a peer evaluation form, their submission will be considered void.**

## 4. AI Declaration

Each student must complete the Moodle AI declaration individually.

- [ ] Complete `AI DECLARATION: ASSIGNMENT 1` in Moodle before the deadline.

## 5. Final submission checklist

Before the deadline, check:

- [ ] GitHub repository is shared with all team members.
- [ ] GitHub repository is shared with `steliosot@msn.com`.
- [ ] One student from the team has uploaded the video to OneDrive.
- [ ] Team video is shared with `s.sotiriadis@bbk.ac.uk`.
- [ ] Team video link is included in the peer evaluation form.
- [ ] Every team member has submitted the peer evaluation form in Moodle.
- [ ] Every team member has completed `AI DECLARATION: ASSIGNMENT 1` in Moodle.
